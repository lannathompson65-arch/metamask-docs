The SDK maintains login state between game sessions so users don't have to re-authenticate on every launch. Session restoration is fast (milliseconds) and works transparently.

## How sessions work

Upon login, a unique session key is generated. The user state is encrypted with this session key and stored on the Web3Auth metadata server. The session key is stored securely on the device using the platform's native secure storage.

On game launch the SDK:

1. Retrieves the session key from secure storage.
2. Requests user state restoration from the metadata server.
3. Decrypts the user state with the session key.
4. Falls back to standard login if no active session is found.

## Configuration

Control session duration in your Web3Auth initialization:

```cpp
FWeb3AuthOptions Options;
Options.clientId = TEXT("YOUR_WEB3AUTH_CLIENT_ID");
Options.network = ENetwork::MAINNET;

// Session configuration
Options.sessionTime = 86400 * 7; // 7 days (in seconds) — default

Web3Auth->SetOptions(Options);
```

### `sessionTime`

Session duration in seconds.

| Bound   | Value          |
| ------- | -------------- |
| Minimum | `1` second     |
| Maximum | `86400 * 30` (30 days) |
| Default | `86400 * 7` (7 days)   |

## Dashboard configuration

Session duration can also be managed from the [Embedded Wallets dashboard](https://dashboard.web3auth.io) under **Advanced Settings → Session Management**, which lets you set the default session duration for all users without changing code.
