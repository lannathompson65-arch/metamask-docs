Server-side verification enables you to securely authenticate users on your backend by validating the ownership of a wallet address. When a user logs in, Web3Auth issues a `JWT` (identity token) signed with the `ES256` algorithm that is unique to your application. Your backend verifies this token to confirm the user's identity.

## How it works

1. **User logs in** — Web3Auth generates a signed `idToken` (JWT) upon successful authentication.
2. **Send token to backend** — Your app sends the `idToken` to your server.
3. **Verify on backend** — Your server validates the JWT using Web3Auth's public key.
4. **Authenticate the user** — Based on verified claims (wallet address, email, etc.), create a session or issue your own token.

**Different flows for different wallets:**

- **Social logins:** The token includes the user's public key linked to their wallet.

## Retrieve the identity token in Flutter

After a successful login, call `getUserInfo()` to retrieve the user's details including the `idToken`:

```dart
final userInfo = await Web3AuthFlutter.getUserInfo();
final idToken = userInfo.idToken;

// Send to your backend for verification using http or dio
final response = await http.post(
  Uri.parse('https://your-backend.com/api/auth'),
  headers: {'Content-Type': 'application/json'},
  body: jsonEncode({'idToken': idToken}),
);
```
